# Checklist Klarifikasi Vendor CCTV Analitik
### Meeting Klarifikasi Sebelum Penetapan Vendor

**Vendor:** _______________________  **Tanggal Meeting:** _______________________

---

## 1. Kapabilitas Teknis (CCTV Analitik)

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 1.1 | Jenis analitik apa saja yang sudah pernah diimplementasikan dan terbukti berhasil di lapangan (deteksi APD, zona bahaya, kendaraan/alat berat, blind spot, dll)? Boleh minta contoh kasus nyata? | | |
| 1.2 | Berapa tingkat akurasi deteksi sistem Anda pada kondisi lapangan tambang seperti berdebu, hujan, minim cahaya, dan getaran alat berat? Ada data pengujian yang bisa ditunjukkan? | | |
| 1.3 | Apakah proses analitik dijalankan langsung di perangkat (edge/on-premise) atau harus mengirim data ke cloud terlebih dahulu? Bagaimana jika koneksi internet di lokasi terputus? | | |
| 1.4 | Apakah sistem Anda bisa diintegrasikan dengan command center/VMS yang sudah kami gunakan saat ini? Jika ya, bagaimana caranya? | | |
| 1.5 | Jika ke depan kami ingin menambah jumlah titik kamera, apakah sistem ini bisa diperluas tanpa perlu mengganti seluruh infrastruktur yang sudah dibangun? | | |

---

## 2. Implementasi & Instalasi

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 2.1 | Berapa lama waktu yang dibutuhkan untuk instalasi satu titik kamera, dan berapa total waktu untuk menyelesaikan seluruh titik sesuai skema penempatan kami? | | |
| 2.2 | Infrastruktur pendukung apa saja yang dibutuhkan (listrik, jaringan/fiber, tiang/mounting), dan siapa yang bertanggung jawab menyediakannya — kami atau vendor? | | |
| 2.3 | Bagaimana rencana Anda agar proses instalasi tidak mengganggu jalannya operasional tambang? | | |
| 2.4 | Apakah Anda pernah mengerjakan proyek serupa di lingkungan tambang? Bisa berikan referensi klien yang bisa kami hubungi langsung? | | |

---

## 3. Layanan & Dukungan Pasca-Implementasi

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 3.1 | Jika terjadi kerusakan atau downtime kamera, berapa lama waktu maksimal tim Anda merespons dan menyelesaikan perbaikan (response time & resolution time)? | | |
| 3.2 | Apakah Anda memiliki teknisi yang berlokasi dekat dengan site kami, atau harus didatangkan dari kota lain? Berapa lama waktu tempuhnya? | | |
| 3.3 | Apakah Anda menyediakan jadwal maintenance rutin/preventif, atau hanya menangani perbaikan setelah ada laporan kerusakan? | | |
| 3.4 | Apakah spare part (kamera, kabel, unit pendukung lain) tersedia di lokasi atau wilayah terdekat, atau harus dipesan dan menunggu pengiriman? | | |

---

## 4. Komersial

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 4.1 | Bisa dijelaskan rincian struktur biaya: mana yang termasuk biaya awal (hardware & instalasi) dan mana yang bersifat biaya berjalan (lisensi analitik, maintenance, support)? | | |
| 4.2 | Berapa estimasi Total Cost of Ownership (TCO) untuk 3–5 tahun ke depan, bukan hanya harga di tahun pertama? | | |
| 4.3 | Bagaimana skema pembayaran yang ditawarkan — apakah bisa disesuaikan dengan pencapaian milestone proyek? | | |
| 4.4 | Apakah ada klausul penalti jika SLA yang dijanjikan tidak terpenuhi? Seperti apa bentuknya? | | |

---

## 5. Keamanan Data & Kepatuhan

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 5.1 | Di mana data video dan hasil analitik akan disimpan, dan berapa lama masa retensinya? | | |
| 5.2 | Siapa saja yang memiliki akses ke data tersebut? Apakah tim vendor bisa mengakses live feed kami? | | |
| 5.3 | Apakah sistem Anda sudah memenuhi regulasi data/privasi yang berlaku untuk industri kami? | | |
| 5.4 | Bisa dipastikan bahwa seluruh data yang dihasilkan sepenuhnya menjadi milik perusahaan kami, bukan milik vendor? | | |

---

## 6. Legalitas & Kapasitas Perusahaan

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 6.1 | Bisa ditunjukkan dokumen legalitas usaha, NPWP, dan sertifikasi relevan (K3, ISO, dll) yang dimiliki perusahaan Anda? | | |
| 6.2 | Bagaimana kondisi finansial perusahaan Anda saat ini, dan apakah cukup stabil untuk menjamin keberlangsungan proyek ini? | | |
| 6.3 | Berapa banyak proyek lain yang sedang Anda tangani saat ini? Apakah tim Anda memiliki kapasitas cukup untuk mengerjakan proyek kami secara bersamaan? | | |

---

## 7. Exit Strategy / Ketergantungan

| No | Pertanyaan | Jawaban Vendor | Catatan / Red Flag |
|----|---|---|---|
| 7.1 | Jika kontrak dengan Anda berakhir, apakah sistem ini bisa diambil alih dan dioperasikan oleh tim internal kami atau vendor lain? | | |
| 7.2 | Apakah kami akan mendapatkan hak akses ke source code atau file konfigurasi sistem jika terjadi perpindahan vendor di kemudian hari? | | |
